# AI股探 交易應用程式 (Trading App)

獨立、在本機執行的交易程式，提供瀏覽器介面（非純終端機輸入）。它會登入您的 AI股探 帳號以接收
即時交易訊號（`BUY` / `SELL`），並透過富邦 Neo API 自動送出對應的委託單。`NOTIFY` 訊號僅供通知，
不會觸發下單。

僅限 **399 方案** 與 **799 方案** 使用者下載。

## 安全性

- 富邦登入資訊、密碼、憑證密碼**只會存在本機記憶體中**，絕不會傳送到 AI股探 伺服器。
- 本機設定檔（`~/.stock-notifier-trader/config.json`）只保存非機密欄位（富邦帳號、憑證檔案路徑，用於下次開啟時預先帶入表單），密碼與憑證密碼一律不寫入磁碟，每次啟動都需重新輸入。
- AI股探 帳密僅用於登入伺服器換取一個短期 JWT，讓程式可以訂閱屬於您帳號的即時訊號；此 JWT 也不會寫入磁碟。
- 本機介面只綁定在 `127.0.0.1`，同網段的其他裝置無法連線。

## 需求

- Python 3.9+，並安裝富邦官方 SDK：`pip install -r python/requirements.txt`
- 富邦證券帳戶、Neo API 使用權限、憑證檔案 (.pfx) — **真實帳戶或富邦模擬交易帳戶皆可**（見下方）

> **注意**：`python/fubon_bridge.py` 中呼叫富邦 SDK 的類別/方法名稱（`FubonSDK`、`Order`、`BSAction` 等）
> 是依據富邦官方 Neo SDK 文件當時的介面撰寫，使用前請對照 [富邦官方文件](https://www.fbs.com.tw/TradeAPI/)
> 確認目前版本的實際介面是否一致，如有落差請直接調整 `fubon_bridge.py`。

## 模擬交易說明

勾選「使用富邦模擬交易環境」後，整個富邦登入欄位區塊（帳號、密碼、憑證路徑、憑證密碼）會被
**隱藏**，完全不需要輸入 —— 程式會自動使用內建的測試帳號：

- 富邦帳號：`41610792`　密碼：`12345678`
- 憑證檔案：`test-cert/41610792.pfx`　憑證密碼：`12345678`
- `python/fubon_bridge.py` 登入時會改用 `FubonSDK(url="wss://neoapitest.fbs.com.tw/TASP/XCPXWS")`
  連線到富邦模擬環境端點，而非預設的正式環境。

之後產生的交易紀錄會標記來源為「模擬」而非「實盤」，方便在本機紀錄與網站儀表板上分辨。

`test-cert/` 資料夾刻意放在 `public/` 之外，避免這份（測試用）憑證被本機介面的靜態檔案伺服器
以 HTTP 提供。

## 安裝與執行

```bash
npm install
pip install -r python/requirements.txt

npm run dev      # 開發模式（tsx，不需先 build）
# 或
npm run build && npm start
```

啟動後會自動開啟瀏覽器（預設 `http://127.0.0.1:4321`），並顯示登入表單：

1. AI股探 帳號密碼
2. 若使用模擬交易，勾選「使用富邦模擬交易環境」— 富邦欄位會整個隱藏，不需要輸入任何富邦資訊
3. 若使用真實帳戶交易，輸入富邦帳號（身分證字號 / 統一編號）、密碼、憑證檔案路徑（送出前會先確認檔案存在）與憑證密碼

點擊「連線並開始監聽訊號」後，程式會持續監聽您帳號底下所有 `actionType: "trade"` 規則觸發的
`BUY` / `SELL` 訊號 —— 但**不會自動下單**。

**確認連線是否成功**：AI股探 帳密與富邦登入成功，只代表準備工作完成；訊號 WebSocket 是否
真的連上是非同步、稍晚才知道的。已連線畫面會多顯示一行即時狀態：
- 🟡（閃爍）「訊號 WebSocket 連線中…」— 尚未確定
- 🟢「訊號 WebSocket 已連線，開始監聽交易訊號」— 確認已在監聽
- 🔴「訊號 WebSocket 連線中斷…」— 連線掉了，socket.io 會自動重試

此狀態每 2 秒更新一次。

## 下單前確認（重要）

每個交易訊號觸發時，程式只會先建立一筆「待確認交易」，顯示在介面的「待確認交易」區塊，包含
方向（BUY/SELL）、股票代號、**委託股數**、觸發價格與規則說明。您必須手動點擊：

- **確認下單** — 才會真正以市價單透過富邦 Neo API 送出委託。
- **拒絕** — 不會下單，該筆訊號會記錄為「REJECTED」狀態，只留紀錄不影響帳戶。

在您做出選擇之前，該筆訊號不會有任何委託送出。待確認清單每 2 秒更新一次。

**委託股數由 AI 規則本身決定**（規則程式碼回傳的 \`quantity\` 欄位，預設 1 張 = 1000 股，
使用者建立規則時可指定張數/股數），不再是本程式固定的環境變數。

介面下方的「交易活動」表格會即時顯示每筆確認/拒絕/下單結果。

點擊「中斷連線」可安全登出並停止監聽；直接關閉終端機（`Ctrl+C`）也會自動登出。

## 交易活動紀錄

每次確認下單（成功或失敗）或拒絕都會：

- 寫入本機紀錄檔 `~/.stock-notifier-trader/activity.log`（JSON Lines），介面的「交易活動」表格即讀取自此檔案。
- 回報到 AI股探 伺服器（只傳送交易結果，不含富邦帳密），可在網站的監控儀表板看到最近的交易活動。

狀態欄位：`FILLED`（真實成交）、`SIMULATED`（模擬成交）、`FAILED`（下單失敗）、`REJECTED`（使用者拒絕，未送出委託）。

## 環境變數 (`.env`)

```
SERVER_URL=http://localhost:3001   # AI股探 伺服器網址；正式環境請改為您的伺服器網址
PORT=4321                          # 本機介面的連接埠
PYTHON_BIN=python                 # 若系統的 Python 執行檔名稱不同（如 python3）
```

`SERVER_URL` 是給部署此程式的人（例如 IT / 維運人員）設定的操作參數，不會出現在使用者介面中，
一般使用者只需要輸入 AI股探 帳號密碼與富邦帳戶資訊。
